05 — LangGraph Orchestration

Autonomous planning,
not hardcoded routing.

The orchestrator is a planning agent, not a router. It calls GPT-4o with the query and portfolio state, receives a JSON plan (agent_plan), and stores its reasoning (orchestrator_reasoning) in state for the frontend to display.

A plan_checker_node runs after every agent. It reads sources_used to know what has completed, and routes to the next unexecuted planned agent — or to END if the plan is complete or a critical node failed.

The graph compiles with interrupt_before=["risk_monitor"], enabling LangGraph's built-in HITL checkpoint. The risk agent only executes after PM approval via the /api/risk/acknowledge endpoint.

⚠ Critical Design Decision — No DataFrames in LangGraph State

LangGraph's MemorySaver cannot reliably pickle pandas DataFrames. All DataFrames live in a module-level DATAFRAME_CACHE dict keyed by session_id. State holds only JSON-serialisable scalars, dicts, and lists. This is a non-obvious architectural constraint that took explicit design to solve.

Scoring Formula

  base_score          # GPT-4o-mini: 0–10
  + tier_bonus        # +2 / +1 / 0
  + portfolio_match   # +1 if ticker in portfolio
)

Why Not X (Twitter)?

X API Basic = $100/month with severe rate limits. X API Standard commercial = $12,000/year minimum. For a prototype targeting free-tier sources, this is not viable.

Production note: In production, X API monitors verified policy accounts. For the prototype, Reddit + Finnhub + SEC EDGAR + Stocktwits provide equivalent coverage at zero cost.

07 — Human-in-the-Loop Design

The machine pauses.
The human decides.

AlphaDesk uses LangGraph's native checkpoint-and-resume mechanism to freeze graph execution the moment a risk limit is breached. The PM sees a live banner, reviews the alerts, and approves or rejects — the graph resumes only after human sign-off.

⚠ Risk Breach Detected — Graph Paused
1
Risk Monitor agent checks metrics against thresholds. max_drawdown = 0.17 exceeds limit of 0.15.
2
LangGraph fires interrupt_before=["risk_monitor"]. Graph execution freezes. thread_id stored in Redis via HITL_PENDING_KEY.
3
Frontend banner polls /api/risk/hitl-status every 10s (adaptive: 120s idle, 10s active). No graph re-runs during polling.
4
WhatsApp notification sent via Twilio: "⚠ AlphaDesk: Risk breach detected — drawdown exceeds limit. Review required."
✓ PM Approves — Graph Resumes
1
PM clicks Approve or Reject on the Risk Monitor page. POST /api/risk/acknowledge with thread_id and approved: true/false.
2
Backend calls resume_graph(thread_id, human_approved). LangGraph restores full state from MemorySaver checkpoint and continues.
3
Decision stored in APPROVAL_HISTORY_KEY (last 50 entries, 30-day TTL). Full audit trail visible in the Approval History panel.
4
HITL Redis flag cleared. Banner disappears. Graph continues to report generation. PM receives PDF tearsheet with HITL decision recorded.
08 — Security & Guardrails

Responsible AI is
non-negotiable.

Every LLM output in AlphaDesk passes through a 3-stage content filter before reaching the user. No raw model output is ever surfaced directly.

3-Stage Content Filter
Every OpenAI output passes through run_content_filter() before returning to the user. Stage 1: keyword/pattern block (investment advice, insider trading language). Stage 2: LLM self-evaluation ("Is this appropriate for a PM research platform?"). Stage 3: audit log with rule_id for every blocked response.
Pattern Matching LLM Self-Eval Audit Logging Rule IDs
Full Audit Trail
Every metric shown has an AuditTooltip displaying its source and computation method. Every AI narrative shows a Disclaimer component. Every risk alert includes triggering_signal.url — a direct link to the SEC filing or news article that caused the alert.
Source Attribution Signal URLs HITL History LangSmith Traces
Configurable Risk Limits
PM can configure all 5 risk thresholds via the UI. Input validated server-side with strict ranges before Redis write. Defaults: Max DD 15%, Volatility 25%, VaR 2.5%, Beta Max 2.0, Correlation Alert 0.75. All limits enforced at graph runtime, not UI level.
Server Validation Range Enforcement Persistent Redis
LLM Cost Controls
Signal scoring uses gpt-4o-mini (10× cheaper than gpt-4o) — the only task where full reasoning is not needed. LangSmith callbacks accumulate token cost in run_cost_usd throughout the run. Full run target: under $0.30. Cost badge displayed in the UI on every run.
Model Routing Cost Tracking LangSmith Callbacks
Graceful API Failure
Every external API call has a stale-cache fallback. If Finnhub is unavailable, the last cached signals are served with an amber system_warnings banner noting cache age. The system never returns a blank screen — GracefulError component shows completed steps and fallback data on any node failure.
Redis Fallback Warning Banners GracefulError UI
Offline Demo Mode
DEMO_MODE=true activates at the top of every agent node, serving pre-recorded JSON snapshots from a real run. Zero API calls, zero network dependency. The live presentation cannot fail due to external service outages. Demo cache is stored in backend/demo_cache/ and loaded by demo_cache_service.
Pre-recorded JSON Zero Network Env Flag

Scalability Architecture
Redis cache-first strategy — no redundant graph runs for cached data
Celery Beat scheduled runs — compute happens at 5 AM, not on user request
DataFrame cache (module-level) — avoids re-fetching within a session
48h cache TTL with data-date freshness check — serves instantly when fresh
gpt-4o-mini routing for high-volume, low-complexity tasks
Early-stage strategy: for a small PM user base, API models remain cheaper than training/serving a custom model
Scale-stage strategy: once request volume is stable, fine-tune a finance model (example: Hugging Face FinGPT family, such as FinGPT/fingpt-forecaster_dow30) for repetitive tasks
Cost reasoning: fine-tuning has fixed costs (dataset curation, training runs, evaluation, MLOps, GPU hosting); with low traffic, those fixed costs are usually higher than pay-per-token GPT usage
Hybrid routing plan: keep complex reasoning on frontier GPT models, shift narrow high-frequency tasks (signal classification, templated summaries) to the fine-tuned finance model to reduce blended cost

12 — Reliability & Responsible AI

The system degrades
gracefully. Never silently.

Failure Hierarchy
1
API failure: serve stale Redis cache + amber warning banner. Never a blank screen.
2
Non-critical node failure: plan_checker logs warning, clears error, continues to next planned agent.
3
Critical node failure (data_ingestion): plan halted, GracefulError shows completed steps and fallback data.
4
Complete outage: DEMO_MODE=true serves pre-recorded real data. Zero network dependency.
Responsible AI Principles
Every AI output labelled with "AI-generated" disclaimer component
No investment advice — system provides analysis, PM makes decisions
All metrics sourced and auditable — AuditTooltip on every number
HITL required for risk-breach decisions — no autonomous trading
Retail signals (Reddit) explicitly labelled as context, not primary signals
Content filter blocks insider trading language, price targets, buy/sell advice

Platform Vision

AlphaDesk is designed as a platform, not a product. Each agent is independently upgradeable. New data sources become a new Tier-1 signal tool. New analytics become a new agent node. The orchestrator routes intelligently to whatever is available.

The end state: a fully autonomous PM research assistant that operates around the clock, surfaces only the highest-signal information, and always pauses for human judgment at the decision point.

10 — Live Deployment

Deployed on Google Cloud + Vercel.

AlphaDesk is fully deployed across a production-grade cloud stack. The frontend is served globally via Vercel CDN, the backend API runs on Google Cloud Run with auto-scaling, and the database is managed PostgreSQL on Cloud SQL.
