Engineering Highlights

Key Technical Innovations

Non-obvious engineering decisions that make this system production-ready and cost-efficient.

Embedding-Based Competitor Ranking (Token Optimization)

Instead of sending all 20 discovered competitors to GPT-4o-mini (expensive, slow), the system first embeds each competitor's basic profile using text-embedding-3-small (~$0.0001), computes cosine similarity against the business context vector, and selects only the top 5. The LLM only sees highly relevant competitors — reducing token usage 4x and eliminating timeouts.

# Rank competitors by semantic relevance BEFORE expensive LLM enrichment def _rank_by_relevance(places, business_context, top_n=5): competitor_texts = [_competitor_to_text(p) for p in places] all_texts = [business_context] + competitor_texts resp = client.embeddings.create( model="text-embedding-3-small", input=all_texts ) vectors = [item.embedding for item in resp.data] query_vec = np.array(vectors[0]) # Cosine similarity: sim(a,b) = dot(a,b) / (||a|| * ||b||) scores = [ np.dot(query_vec, np.array(v)) / (np.linalg.norm(query_vec) * np.linalg.norm(np.array(v))) for v in vectors[1:] ] ranked = sorted(zip(places, scores), key=lambda x: x[1], reverse=True) return [p for p, _ in ranked[:top_n]]
🛡️

Graceful Degradation

Every LLM-dependent feature has deterministic fallback heuristics. If OpenAI is unavailable, hardcoded defaults kick in — questionnaire templates, rule-based scoring, template personas. Zero downtime.

🗄️

pgvector + psycopg v3

SQLAlchemy connect event registers pgvector type on every new connection. Without this, psycopg v3 silently stores embeddings as NULL — a subtle integration pitfall solved with one event listener.

🌍

Regional API Routing

OpenAI regional API keys require region-specific base URLs (us.api.openai.com). Config validates the base URL at startup and surfaces actionable errors rather than cryptic 401s.

📐

Structured LLM Outputs

All 30+ LLM prompts return structured JSON validated against Pydantic schemas. Malformed responses are caught, logged, and fallback data is returned — ensuring the frontend never receives broken state.

📍

Multi-Stage Geocoding

Address → Geocode → Nearby Search (1km radius, auto-expands to 2km). Falls back to text-based Place Search if geocoding fails — handles partial addresses, rural locations, and API rate limits.

📊

Prometheus Observability

Every FastAPI route is instrumented with Prometheus metrics — request counts, latency histograms, OpenAI call tracking. Rate limiting via slowapi prevents abuse on public endpoints.

RAG Retrieval Evaluation

After each semantic memory retrieval, computes per-chunk cosine similarity between query vector and result vectors using NumPy. Logs avg_similarity, min_similarity, and a low_relevance flag when min similarity drops below 0.75 — enabling detection of poor-quality RAG retrievals.
